# ADR-008: Explicit Source Adapters and Bounded Evidence Retention

## Status

Accepted

## Date

2026-07-15

## Context

No production portal or agency contract has been approved. The MVP must avoid
speculative integrations, preserve field-level evidence, support later audits,
and minimize retention of third-party source content.

## Decision

Define a deterministic `HouseSearch.Ingestion.Adapter` behaviour and
fixture-backed reference adapters. Add a production adapter only after a named
source has recorded terms, robots, permitted method, host allowlist, rate limit,
and city coverage approval. The MVP permits official APIs, feeds, and static
HTTP retrieval. Browser automation and LLM extraction are outside scope.

Persist immutable normalized listing snapshots and field-level provenance for
as long as the related case history is retained. Store a response hash,
retrieval metadata, and sanitized raw response for operational diagnosis. Delete
the sanitized raw response after 30 days while retaining normalized evidence
and its hash. Never retain credentials, cookies, unrelated personal data, or
unbounded page assets.

## Alternatives Considered

### Alternative 1: Generic configurable HTML scraper

- **Description**: Let administrators define selectors and scrape any approved site.
- **Pros**: Faster apparent source onboarding.
- **Cons**: Weak contracts, fragile parsing, and unsafe configuration surface.
- **Why rejected**: Source-specific deterministic adapters are more auditable.

### Alternative 2: Retain full raw responses indefinitely

- **Description**: Preserve every source body for complete replay.
- **Pros**: Maximum debugging history.
- **Cons**: Storage, licensing, privacy, and no-republishing risk.
- **Why rejected**: Normalized immutable evidence is sufficient for recommendation audit.

## Consequences

### Positive

- Source authorization precedes implementation and retrieval.
- Contract tests can run without live portals.
- Recommendation facts remain auditable after raw payload deletion.

### Negative

- Each production source needs adapter code and fixtures.
- Full parser replay is unavailable after 30 days.

### Risks

- Sanitization may miss sensitive content; adapters must allowlist retained fields.
- Deleted raw payloads may hinder late debugging; hashes and normalized evidence mitigate this.

## Implementation Notes

Use Finch through a source-aware client enforcing HTTPS, approved hosts,
redirect validation, timeouts, response-size limits, and rate limits. Run a
daily Oban retention job that nulls or deletes expired raw bodies without
removing snapshot evidence.
